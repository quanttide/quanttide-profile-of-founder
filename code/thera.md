# 苹果AI外脑

## 语境

default activity：默认活动模拟和辅助人类的默认神经网络。收集和处理日常想法流动的最小 CODE 循环。领域为思考。
1. 收集： 润色记录和确认，以文本的形式存。
2. 组织：打分类标签。积累到一定数量再迭代。
3. 提取：同类标签积累到一定程度再迭代。
4. 表达：提取量到一定程度再迭代。以想法是不是可执行、是不是感觉到放松感作为思考完成的标准。

fiction activity：小说创作活动。从情绪日记出发形成创作素材，再从创作素材形成小说片段，再从小说片段组合成正文。领域为思考+写作。

thera activity：AI外脑编程活动。领域为思考+编程。

think domain：上述活动中共用的关于思考的程序性记忆和陈述性记忆。
write、code domian：待进一步发现。
domian 的核心价值在于单个 activity 无法获得的元方法进行迁移。
asset->infra 连接底层基建
state->metadata 系统元数据
activity 再上一级是 business。因为个人的“业务”比较小，所以暂时使用“activity”，看看哪一层建模更合适。

## 计划

根据“能让 AI 做的就不要人做”这一宪法，您的开发优先级应该发生根本性的逆转。不要先去打磨核心算法，而是先去封死那些让您在做“苦力”的缺口。

优先级 P0：输入端的“零摩擦封装”
痛点：您每天产生大量碎片，如果还要手动整理、打标签、归档，这就违反了原则。 封装目标：“吞噬一切”的接收器。
* 功能：一个极简的前端（甚至是一个语音转文字接口、一个浏览器插件），只负责把数据扔进“思考云”。
* 自动化逻辑：
    * 人只负责“吐”数据。
    * AI 负责：自动清洗、自动打标签（提取实体）、自动向量化、自动存入知识云。
* 收益：您只需“说”，不用“管”。这是系统的燃料供给系统，必须先通。

优先级 P1：输出端的“硬约束封装”
痛点：您不敢放手让 AI 去做，因为还要花大量时间去检查它的幻觉和错误。 封装目标：“铁面无私”的校验器。

优先级 P2：中间层的“路由协议封装”
痛点：您还在手动决定“这个想法该变成代码还是文章”。 封装目标：“语义共振”路由器。
* 功能：基于向量相似度的自动分发。
* 自动化逻辑：
    * 接收 P0 的数据 -> 计算向量 -> 广播 -> 哪个执行云（P1 封装好的）响应就给谁。
* 收益：您不用再关心“流程”，您只面对“结果”。
