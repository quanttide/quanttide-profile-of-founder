# 新媒体平台开放API（待验证）

一、准入主体（谁能申请）

1. 微信（公众号/开放平台）

• 个人：可注册订阅号/小程序，基础接口可用

• 企业/个体：全量接口开放（服务号、支付、卡券等）

2. 微博开放平台

• 个人+企业均可，无严格限制，准入最宽松

3. B站开放平台

• 个人+企业均可，支持个人工具开发

4. 小红书开放平台

• 仅企业/机构，个人/个体不可申请

5. 知乎开放平台

• 仅企业/机构，个人不可申请

6. 抖音开放平台

• 个人：仅可创建小游戏/小玩法，不可做小程序/内容发布

• 企业：可创建小程序/移动/网站应用，需注册资本≥50万、注册≥1年

• 个体工商户：不开放发布能力

二、接口能力（核心：能不能发内容）

1. 微信

• 可读+可写，能力最完整

• 支持：图文发布、素材管理、菜单、支付、小程序、定时发文

2. 微博

• 可读+可写，开放度高

• 支持：发博/编辑/删除、评论、私信、直播、话题、榜单

3. B站

• 可读+可写，全功能开放

• 支持：视频发布/编辑/定时、稿件管理、互动数据、直播

4. 小红书

• 仅只读，无写入

• 仅查笔记、评论、数据；不支持发布/编辑/删除

5. 知乎

• 几乎只读，写入极少

• 仅查回答、文章、数据；无公开发文接口

6. 抖音

• 可读+有限写入，审核严格

• 支持：SDK投稿（拉起抖音发布）、H5分享、视频数据查询、私信管理（限高门槛）

• 无纯API自动发文：必须用户手动确认发布

• 发布能力仅开放：创作工具/社区/硬件/游戏类企业应用

三、开发场景与合规边界（避坑指南）

✅ 适合「内容发布/自动化工具」

• 微信、微博、B站：支持API发文、批量管理、定时发布

• 抖音：仅支持SDK拉起发布（用户手动确认），无纯后台自动发文

• 小红书、知乎：绝对禁止，无合规接口

✅ 适合「数据分析/内容统计/榜单」

• 六大平台均可：均开放数据查询接口

❌ 绝对禁止（红线）

• 小红书/知乎：自动发文、批量发笔记、自动回答（违规）

• 抖音：纯后台自动发文、无用户确认发布（违规爬虫）

• 所有平台：爬取非公开数据、恶意刷量、滥用接口（易封号）

四、一句话核心结论（必记）

1. 微信、微博、B站：个人/企业均可，可读可写，支持全链路内容管理

2. 抖音：企业准入、有限写入，仅支持SDK拉起发布，无纯API自动发文

3. 小红书、知乎：仅企业准入，只读不写，仅适合数据类开发

4. 合规唯一路径：仅微信、微博、B站有官方纯API发文接口；抖音需用户手动确认
