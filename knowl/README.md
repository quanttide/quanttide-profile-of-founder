# 知识

本模块用于结构化存储业务概念的**本体**（抽象规则）与**实例**（具体对象），实现知识模型的清晰管理、可扩展性及团队协作效率。通过"本体-实例分离"的设计，避免框架与内容混用，确保知识模型的稳定性和可维护性。

## 目录结构

```
knowl/
├── ontology/          # 本体定义（抽象规则）
│   └── brand.yaml      # 品牌类本体
└── instance/          # 实例数据（具体对象）
    └── brand_founder.yaml  # 量潮创始人实例
```

## 本体（Ontology）维护指南

本体是抽象类的通用规则，定义了领域的核心概念、属性类型、关系类型及约束。修改本体需谨慎，避免破坏实例的一致性。

### 修改场景

- 新增业务概念（如新增 Product 类）
- 调整属性类型（如为 Brand 类添加 business_distribution 枚举选项）
- 优化关系类型（如新增 aggregates 关系）

### 操作步骤

1. 打开 `knowl/ontology/` 目录下的目标本体文件（如 `brand.yaml`）
2. 按照 YAML 格式修改类定义
3. 保存文件后，检查实例是否符合新规则

### 注意事项

- 本体仅定义属性类型（如 enum、boolean），不包含具体值
- 修改本体前，需评估对现有实例的影响

## 实例（Instance）维护指南

实例是抽象类的具体表现，填充了属性值、关系目标及维护细节。实例的修改不影响本体，确保灵活性。

### 添加新实例

1. 在 `knowl/instance/` 目录下新建 `.yaml` 文件
2. 填写实例的基本信息（参考示例格式）

### 修改现有实例

直接编辑目标实例文件，修改属性值或关系目标即可。

### 注意事项

- 实例名称需唯一且符合业务语义
- 关系目标的 target 需指向已存在的实例

## 常见问题

### 本体中找不到实例的类？

检查实例的 class 字段是否与本体中的类名一致。

### 实例的属性值不在本体枚举中？

先修改本体，再更新实例。

### 实例的关系目标不存在？

确认目标实例是否在 `knowl/instance/` 目录下。

## 总结

本模块通过"本体-实例分离"的设计，实现：

- **清晰性**：框架与内容职责分明
- **可扩展性**：新增业务概念时，只需添加实例
- **协作性**：团队可并行维护本体与实例
