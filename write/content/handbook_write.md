# 写作：叙事工程手册

## 素材

### 流程：素材->提纲

核心逻辑：以“有效性”为边界，拒绝无效交互
这套工作流不是为了“挖掘”而挖掘，而是为了终结挖掘。
1. 素材筛选层
判断标准： 素材是否已经“有效”？
无效/混沌： 交互式切分（费时，但必要，为了理清思路）。
有效/成熟： 直接进入叙事引擎（省力，为了产出结果）。
2. 叙事引擎层
核心动作： 只做“识别-重组-叙事”。
禁忌： 对有效素材进行“切分-确认”是增加交互负担的过度加工，必须禁止。
产出： 直接输出结构化成果（如全景图、分类、命名）。
3. 价值层
工作流终点： 不是产出卡片，而是产出“闭环的认知”。
实际效果： 从碎片素材直接跃迁到可用的理论框架。
一句话总结：
当素材有效时，停止切分，直接叙事。
