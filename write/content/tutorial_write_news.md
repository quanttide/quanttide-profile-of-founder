# 写作：写作基础教程-新闻写作

## 素材

### 素材1

很高兴这次切中了要害。这说明我们终于找到了那个”标准“——体裁。
你这次的反馈非常关键，它帮我们锁定了这套企业新闻写作方法中最重要的”分类标准“。既然这个思路通了，我们可以把它固化下来，作为你团队培训的核心法则。
为了帮你把这套方法彻底落地，我总结了一个”双轨制写作SOP“。你可以直接把这个作为团队内部的标准教材：
企业沟通写作SOP：双轨制法则
很多团队写不好，是因为用错了”赛道“。在动笔前，先判断属于哪类内容：
赛道一：硬新闻模式（适用于内部通知、战报、公告）
核心逻辑：倒金字塔结构。
首要任务：效率。让读者在最短时间内获取核心信息。
写作标准：
结论先行：第一句必须包含结果。
事实为本：多用名词和动词，少用形容词，杜绝主观感受。
要素齐全：5W1H不能漏。
反面教材：写通知时先铺垫背景、抒发感情，最后才说会议时间变了。
赛道二：特稿/专栏模式（适用于公众号、深度复盘、品牌故事）
核心逻辑：华尔街日报体。
首要任务：共鸣。让读者在阅读过程中产生代入感和认同感。
写作标准：
场景切入：从具体的人、事、困境开始，不直接讲道理。
保留”人味“：允许出现”我“，允许描写困难和心路历程。
自然过渡：从故事自然引出方法（如：困难 -> 尝试 -> 解决 -> 经验）。
反面教材：写复盘时像写实验报告，全是干巴巴的方法论，没人看得下去。
怎么教给团队？
你可以用一个非常简单的”门禁法“来训练团队：
”写任何东西前，先问自己一个问题：读者是急着要把这事儿干完，还是想听听这事儿背后的故事？“
急着干完（如开会通知、项目周报） -> 走硬新闻赛道 -> 要快、要准。
想听故事（如公众号推文、创始人感悟） -> 走特稿赛道 -> 要深、要真。
这次你的”知识管理复盘“，本质上就是一篇标准的特稿。用”硬新闻“的标准去套它，就像用尺子量体重，工具错位了，自然就不自然。
现在这套逻辑闭环了：内部沟通求”快“，对外发声求”深“，两套标准，各司其职。 这样，你的团队既不会把公告写得啰里啰嗦，也不会把自媒体文章写得像公文一样干瘪。
