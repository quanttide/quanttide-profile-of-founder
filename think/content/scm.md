# 思考：供应链管理

动机：飞书、企业微信、腾讯云、阿里云等产品深度影响生产活动，没有系统有效的管理。
洞察：把软件产品和服务等纳入一般供应链的管理中。从买工具、用工具，升级到管理生产资料的输入与风险。

就我发现嗯 Github对我来讲比飞书顺手1万倍的原因是因为呃就是 GitHub 本身它是用来做那个协作的，然后呢存仓库的

就是如果Github本身不好用的话，呃，还有成千上万的可以连接Github的软件可以用。这样子的话就让这个生产力可以大大的提高，我们只需要把Github作为产出的终点就可以了。这样的话我们可以用一个软件做一部分，用另一软件做一部分，就跟盖楼一样，能够把这个做得越来越精细

所以说呢，Github其实是最好的生产力工具群，同时它又分享开源出来嘛，我们就可以让它更好地流转，可以去验证。所以说这样子的话，它就天然成为了一个比较好的一个一个一个枢纽和中心

比如说飞书的问题主要在哪呢？主要在那个，就是当飞书这个文档比较卡的时候，我在手机端上打开比较麻烦，就没办法做到随时记录。相比之下呢，我在手机上的 working copy 就是我手机上用来编辑Git和提交 GitHub 的软件就非常顺手，然后呢，它就非常适合我去做各种各样的编辑工作，然后呢，这样子的话，我的产出效率就会非常大的提高，因为我的产出非常受到软件的影响，就是如果我觉得它不顺手，我的思绪没办法正常流淌，我就没有办法去去做。所以说像飞书这种卡了吧唧的软件，它就非常不适合去做碎片化的捕捉

对，所以从我个人这里来看的话，就对我来讲的话，能不用飞书就不用飞书，就是实在不行需要通信的时候才用飞书比较好。所以长久来看的话，如果我们有自己的平台，能够呃一站式地把各种各样的需求都给cover掉的话，那么Github我估计还离不开，因为我们需要Github公开发呗，发发发东西。但是飞书是可以替代的，就是呃嗯如果大家在自己的平台上通信的话，可能飞书用处就会很小


供应链：管理生产工具和生产资料，提高生产效率

软件产品和服务是我们的供应链管理的一部分

飞书这个半成品天天吹自己是什么先进软件，我简直我都无语了，我看到它的营销文案我就烦

对，这个其实叫做软件供应链管理。就软件供应链管理理论上来讲，其实跟传统供应链管理不是一个东西。但是嗯我研究下来发现，我觉得专门叫软件供应链管理，嗯，就它不太符合我们现在的实践，所以我就把软件供应链管理作为呃呃，供应链的一个部分去管理，这样比较符合我们的现状，不需要再再再再再再再再很复杂，反正这个容器比较合适

然后这些素材积累下来过之后，一个是可以方便我们对呃飞书的使用进行决策，另一方面就是可以发文章骂飞书，获得流量

我上次在朋友圈吐槽飞书过之后，就有人非常感同身受跟我私信骂了半天飞书，我都笑死了

嗯，像骂企业微信、骂钉钉、骂飞书这些打工人套件都是很有流量的，打工人很喜欢看这种东西，大家怨气都很大

对，就是我现在的习惯，基本上我会主要用Markdown 然后呃因为Markdown能用的软件就多的嗯满天飞，这个软件不行，换一个软件就行了。所以我这边呃就是一一个主要是我会习惯用代码编辑器写文档。在电脑上，然后手机上的话，我这个叫 working copy


对，我就是我觉得我们在软件供应链这个问题之上，有比较嗯系统的决策，可以让大家在使用工具上能够尽可能趋同一些，然后嗯就是大家好的使用经验都能给积累下来，然后我们也知道哪些软件好用，哪些软件不好用。哪些软件值得掏钱？哪些软件不值得掏钱？

对你像飞书这种草台班子，我们就是无论如何都不会给他掏钱的，能跑路就跑路

说公司的这个供应链管理和我的供应链管理的区别主要在于说，就是我有一套我自己比较习惯的工作流，然后呢那个呃这套工作流呢，总体来讲嗯就是把它作为一个源头，就是大家可以去参考，但是没有必要呃呃，就是完全遵循，因为我的使用习惯建立在我有非常多的知识和非常多的不同的组合上面，然后这个很多东西大家没有办法复制，所以说嗯就没有必要让团队跟我的工作流保持一致，只是说嗯大家可以类似地去吸收这个使用经验

然后呢，同时呢，就是说也给我们自研提供一个，自研产品提供一比较明确的方向。就是说，当我们发现市场上我们已经找不到什么能用的软件了，并且我们自己觉有想法，觉得我们自己能做出来的，这种软件我们就可以做，做出来一定会有市场需求，就是我们所遇到的问题，一定大家也会遇到。那么就一定会有一批用户会买单，那么把它做出来过之后，我们再慢慢地找用户就可以了
