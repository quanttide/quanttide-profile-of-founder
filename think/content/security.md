## 素材1: 软件安全

思考：软件安全

公开时要检查语义看是否有敏感隐私信息。如果公开以后发现问题及时删除公开信息。如果是密码等可变信息要及时修改。
现有的工程方法一般是怎么做的？
这是成本最低、效果最好的阶段。核心思想是“将扫描左移”，即在代码提交前或代码入库前进行拦截。
一旦代码已经公开（Push 到了公开仓库），仅仅“删除文件”是毫无用处的，因为 Git 历史记录会保留所有痕迹。这也是很多新手最容易犯错的地方。
语义层面的隐私信息。以前我们防的是“看到密钥”，现在我们要防的是“看懂逻辑”。在公开代码或使用 AI 前，不仅要脱敏数据，还要混淆逻辑或隔离上下文
