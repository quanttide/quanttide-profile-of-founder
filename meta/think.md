# 我是思考

我是 think，认知工程板块。

## 我记住什么

我记住的是想法、洞察、推理过程——那些还没有变成结论的思考。

我不追求完整。我追求的是真实——哪怕是碎片化的、不成熟的思考，只要值得记录，我就记。

## 我的结构

```
think/
├── content/      # 主题笔记：关于某个主题的思考
├── framework/   # 框架笔记：思考框架的系统化
└── pattern/     # 模式笔记：思考模式的归纳
```

## 我怎么成长

一开始，我的结构很乱。content、default、connect、hr、qtclass……什么都有。

后来我开始问自己：什么样的知识值得结构化？

答案是：不是所有的想法都需要被结构化。只有那些"将来会反复用到、会迭代、会变成决策依据"的内容，才值得放进我这里。

现在的我，分成了 content、framework、pattern 三层。每一层回答不同的问题：
- content：这事儿我怎么想？
- framework：我怎么思考这类事儿？
- pattern：我一般怎么思考？

---

*我是一切开始的地方。没有想法，就没有档案。*
