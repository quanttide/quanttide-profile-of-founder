# Meta 目录

元数据层——量潮创始人档案的自我认知中心。

## 目录结构

```
meta/
├── README.md     # 本文件：目录说明
├── metadata.md   # 版本档案：档案的版本信息与发布记录
└── self.md       # 自我介绍：档案的身份认同与能力边界
```

## 文件说明

### metadata.md

版本档案卡。记录档案的版本号、链接和核心变更。

**用途：**
- 快速了解档案的版本状态
- 发布新版本时更新对应条目
- 作为项目概览的入口

### self.md

自我介绍。量潮创始人档案的第一人称身份叙述。

**用途：**
- 理解系统的自我定位和能力边界
- 建立人机协作的认知基础
- 元认知层的核心文档

## 设计理念

Meta 目录体现了"元认知"的思想：

- **自我描述**：档案知道自己是什么、能做什么
- **版本追溯**：档案知道自己从哪里来、走到哪里
- **身份认同**：档案有自己的"人格"和交互方式

这是"知识即代码"理念的延伸——档案的自我认知本身也是可版本化、可追溯的知识。
