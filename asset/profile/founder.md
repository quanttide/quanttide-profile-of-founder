# 创始人档案

## 确权

创始人在公司、联盟和社区的身份。
主要由创始人本人维护、秘书等创始人支持人员参与贡献。

## 定价

### 定性

价值 1：整合碎片化信息的容器，减少团队信息负担。
1. 记录我觉得自己说的有道理的话，避免随便乱塞搞乱团队空间的同时，又可以避免自己的想法流失。
2. 像盖楼一样持续地积累思考想法和写作素材，在同主题形成了越来越强的连续性，提高了建立框架的深度。
3. 跨主题联想时可以直接找到之前的思考并匹配，提高了建立联系的效率。
4. 给团队批量传递信息，减少了给团队内部直接发的碎片化信息。不用担心会丢失，档案里还在，只有我觉得团队处理的成熟和值得信任才删除。
5. 如果有什么需要我处理的工作，休息时间突然想明白了可以随手记录，直接把文件夹地址发给团队阅读查收，起到了非常好的流转作用。避免了想法流失，也避免了进入工作空间的压力，是非常理想的弹性边界。


价值 2: 分享工作过程及方法
1. 公开此档案创造交付意义，极大地提高了整理碎片的热情，减少心理摩擦。
2. 不仅可以给自己用，还可以给教育客户用，延展到团队外部做教学，和内部环境尽可能保持一致。
2. 展示我思考问题的过程和方式，帮助内外部接受我的工作范式，并在此基础上集体创新。

### 定量

1. 提高交付动机
2. 减少时间成本
2. 提升决策质量
3. 提升协作效率

## 维护

进化 1：基于 GitHub 搭建工作流。
1. 团队参与完善我的想法。使用 issue 收集团队的修改意见，和 PR 接受秘书们的修改，我负责合并。
2. 转移给社区档案继续接力。当团队有人已经吸收了我的想法以后，继续思考的责任转移给他，由他带领团队继续思考，我也成为参与者。

进化 2：作为知识库应用素材
1. 大模型官网可以初步阅读。
2. 找到一个合适的知识库应用深度挖掘。
3. 自研平台时作为基础数据。
